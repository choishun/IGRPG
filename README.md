## 작품 소개

IGRUS 동아리에 갓 가입한 신입 부원이 아그둥이 활동을 위해 처음 동방을 방문하면서 벌어지는 이야기를 담은 텍스트 RPG 게임입니다.

동방에서 마주치는 세 가지 문제를 몬스터로 형상화하여 전투로 해결해 나가며 앞으로 행복한 아그 생활을 하는 것이 목표입니다.

### 줄거리
가입비 20000원을 내고 아그둥이를 신청한 주인공.  
설레는 마음으로 동방 문을 열었는데...

```
"어으... 이게 대체 무슨 냄새야!!!"
```

동방을 괴롭히는 존재들을 하나씩 물리쳐 아그 동방을 지켜라!

## 주요 기능

### 턴제 전투 시스템
매 턴 플레이어가 행동을 선택하고 적도 동일한 방식으로 대응합니다.

| 번호 | 행동 | 효과 |
|------|------|------|
| 1 | 공격 | 공격력만큼 데미지 30% 확률로 크리티컬 (2배) |
| 2 | 집중 공격 | 다음 공격에 +30 데미지 추가 |
| 3 | 방어 | 이번 턴 피해 0 연속 사용 불가 (쿨다운) |

- 적의 행동은 위 3가지 중 **랜덤**으로 결정됩니다.
- HP가 0이 되면 전투 종료, 여러 존재들의 가호로 **재도전**할 수 있습니다.

## 신경 쓴 기술적 포인트

### 1. 추후 확장을 위한 해더파일 분리
나중에 기능과 전투시스템을 더 확장시키기 위해 해더파일을 분리하였습니다.

### 2. 참조자(&) 전달로 전투 상태 유지
초기에 `battle_start(player py, monster ms)` 로 값을 복사해서 전달했더니 전투 중 HP 변화가 원본에 반영되지 않는 버그가 있었습니다. 이를 해결하기위해 `player& py, monster& ms` 로 참조자 전달로 변경해 해결했습니다.

### 3. 방어 쿨다운 상태 관리
방어를 연속으로 사용하지 못하도록 `shieldUsed` 플래그로 쿨다운을 구현했습니다. 플레이어와 몬스터 각각 독립적인 쿨다운 상태를 가집니다.

### 4. 몬스터 방어 문제 해결
주인공의 행동 이후 몬스터의 행동이 진행되어 몬스터의 방어 여부와 관계없이 공격이 들어간다는 문제점이 있었습니다. 이를 해결하기 위해 몬스터의 선택을 주인공의 행동전에 방어면 if문으로 미리 예고를 하는코드를 사용해 이를 해결하였습니다. 
 
### 5. GameEngine으로 흐름 분리
`main.cpp`는 `GameEngine::GameRun()` 한 줄 호출만 남기고 스토리 진행/전투 호출/재도전 처리를 모두 `GameEngine`이 담당하도록 역할을 분리했습니다.
